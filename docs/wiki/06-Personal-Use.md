# استفاده‌ی شخصی هم‌زمان با فروش (اپراتور)

می‌خواهی هم به دیگران بفروشی و هم خودت از همان نود مثل یک نود معمولی PasarGuard استفاده کنی؟ دو راه داری.

## راه ۱ (توصیه‌شده): یک مستأجر شخصی برای خودت
یک مشتری/اشتراک با **سهمیه‌ی بزرگ** برای خودت بساز و پنل PasarGuard شخصی‌ات را با کلید همان مستأجر وصل کن. دقیقاً مثل یک مشتری معمولی، فقط حجمش را بالا بگذار.

1. پلن بساز (مثلاً ۱۰ ترابایت، مدت طولانی).
2. مشتری «self» بساز و اشتراک بساز → کلید مشتری بگیر.
3. پنل شخصی‌ات را با آن کلید + گواهی + `NODE_IP:62050` وصل کن.

مزیت: کاملاً ایزوله از کانفیگ هسته، بدون ریسک، همین حالا کار می‌کند.

## راه ۲: مدیریت هسته با core key
اگر می‌خواهی **خودِ کانفیگ هسته‌ی Xray** را از یک پنل PasarGuard مدیریت کنی:
1. نود را با `--core-key` نصب کن (نگاه کنید: [نصب نود](03-Install-Node)).
2. در پنل PasarGuard، نود را با **core key** (به‌جای کلید مشتری) اضافه کن.
3. حالا با core key می‌توانی کانفیگ هسته را push کنی (Start config اعمال می‌شود)، ولی عملیات کاربر/Stop آن نادیده گرفته می‌شود تا چند-مستأجری نشکند.

> هشدار: core key کانفیگ مشترکِ همه‌ی مشتری‌ها را عوض می‌کند. با احتیاط استفاده کن. برای استفاده‌ی شخصیِ صرفِ مصرف، **راه ۱ امن‌تر است.**

## آیا کاربرهای من با کاربرهای مشتری‌ها تداخل می‌کنند؟
نه. هر مستأجر فضای نام مجزا دارد (emailها namespaced می‌شوند) و یکتایی credential (uuid/password) در کل نود enforce می‌شود. داده‌ها و آمار هر مستأجر جداست.
