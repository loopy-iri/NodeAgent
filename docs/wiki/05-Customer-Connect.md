# اتصال از پنل PasarGuard (مشتری)

تو (مشتری) پنل PasarGuard خودت را داری. نودِ اپراتور را مثل یک **نود معمولی** اضافه می‌کنی.

## چیزهایی که اپراتور به تو داده
1. **آدرس نود:** `NODE_IP` + پورت gRPC (پیش‌فرض `62050`)
2. **گواهی نود** (یک بلوک PEM `-----BEGIN CERTIFICATE-----`)
3. **کلید مشتری** (API Key)

## افزودن نود در پنل PasarGuard
در بخش افزودن نود این‌ها را وارد کن:
| فیلد | مقدار |
|------|-------|
| Address / Host | `NODE_IP` |
| Port | `62050` (یا پورتی که اپراتور گفته) |
| Protocol | **gRPC** |
| Certificate | گواهی نود (همان PEM) |
| API Key | **کلید مشتری** (نه master/core key) |

## بعد از اتصال
- نود باید وضعیت **connected** بگیرد. اگر پنل تو کانفیگ هسته بفرستد، نود آن را **نادیده** می‌گیرد ولی خطا نمی‌دهد (چون نود حجمی است؛ کانفیگ هسته دست اپراتور است).
- کاربرهایی که در پنل خودت می‌سازی (vless/vmess/...) روی نود زیر مستأجر تو اضافه می‌شوند و در پنل خودت دیده می‌شوند.
- آمار مصرف هر کاربر در پنل خودت برمی‌گردد.
- با اتمام حجم/انقضای اشتراکت، کاربرهایت قطع و دسترسی کلیدت رد می‌شود. بعد از تمدید توسط اپراتور، دوباره وصل می‌شوی.

## نکته‌ی حیاتی برای کارکرد لینک کاربر نهایی
نود یک inbound واقعی مشترک دارد (مثلاً `vless-in` روی پورت `443`). برای اینکه لینک‌های کاربرانت کار کنند:
- inbound پنل تو باید از نظر **پورت، پروتکل، شبکه (tcp/ws/...)، SNI/host و تنظیمات TLS** با inbound واقعی نود یکی باشد.
- این مقادیر را از اپراتور بگیر و دقیقاً همان‌ها را در پنل خودت تنظیم کن.
- سیستم به‌صورت خودکار کاربرهای تو را روی inbound واقعی نود می‌نشاند (`force-inbounds`)، ولی پارامترهای اتصالی که در لینک تولید می‌شود از پنل توست — برای همین باید بخوانند.

## محدودیت‌ها
- نمی‌توانی کانفیگ/پروتکل هسته را عوض کنی (دست اپراتور است).
- نمی‌توانی نود را stop کنی.
- سهمیه‌ی حجم و انقضا را اپراتور تعیین می‌کند؛ انقضا در سطح اشتراک توست (نه per-user).
